# 伯乐Skill 使用说明模板

这份文档是给维护者自己用的。

目标只有一个：**以后你想让 AI 帮你维护这个项目时，直接复制下面的中文模板即可。**

适用对象：

- Claude Code
- Codex
- Trae
- 其他支持读取仓库上下文的 AI 编程助手

## 先记住一句话

`伯乐Skill` 不是网站运行时必须依赖的在线能力，而是你在“加源、修源、改规则、控成本、排查问题”时给 AI 的工作方法。

也就是说：

- 网站日常自动更新：主要靠 `GitHub Actions + Python 脚本 + RSS/公开页面`
- 你要继续扩展项目：可以把下面的模板发给 AI

## 最短可用模板

如果你只想快速开工，直接复制这一段：

```text
请使用伯乐Skill，先理解这个仓库是一个部署在 GitHub Pages 的 AI 资讯雷达。请优先保持默认公开流程免费、稳定、可自动更新，不要默认依赖付费 API、登录态、cookie 或私有 token。

请先问我要目标和信息源清单，再帮我判断每个信源应该用 RSS、OPML、公开 feed、官方 changelog、静态页面抓取、Jina 兜底、AgentMail，还是应该跳过。

要求：
1. 优先复用仓库现有抓取模式和数据结构。
2. 不要把任何 API Key、cookie、token、真实私有 OPML、邮箱正文、私有邮件内容提交到仓库。
3. 尽量保持 GitHub Actions 和 GitHub Pages 可直接运行。
4. 改动后请做最小必要验证，并告诉我改了哪些文件、为什么这样改。
```

## 模板 1：让我先整理信息源

适合场景：

- 你脑子里有一堆来源，但不知道该怎么接
- 你想让 AI 先帮你分类，而不是上来就改代码

直接复制：

```text
请使用伯乐Skill，帮我整理我的 AI 信息源。

你先不要改代码，先问我要信息源清单，并按下面维度帮我分类：
1. 这个源是否真的长期值得追踪
2. 它更适合用 RSS、OPML、公开 feed、官方 changelog、静态页面抓取、Jina 兜底、AgentMail 还是跳过
3. 它适合做成公共默认源，还是只适合做我的私人高级源
4. 它的稳定性、维护成本、是否依赖登录、是否可能收费
5. 它和现有源是否重复

请最后输出一张清单，分成：
- 建议加入公共默认层
- 建议加入私人 OPML / 高级层
- 暂时观察
- 建议跳过
```

## 模板 2：帮我加一个新源

适合场景：

- 你已经知道某个站想接入
- 想让 AI 从“可维护性”角度接入，而不是写一堆临时脚本

直接复制：

```text
请使用伯乐Skill，帮我把这个新源接入 AI 资讯雷达：

信息源：
- 名称：[这里填名称]
- URL：[这里填网址]
- 我关心它的原因：[这里填原因]

要求：
1. 先判断它最适合的接入方式。
2. 如果能用 RSS / Atom / 官方 feed，就不要写脆弱抓取。
3. 如果必须抓页面，请优先用现有解析模式，避免引入复杂依赖。
4. 判断它应该进默认公开源，还是只放进私人源。
5. 如果不适合接入，请明确告诉我原因，不要强行接。
6. 如果要改代码，请顺手补最小必要测试并跑验证。
```

## 模板 3：帮我修失败源

适合场景：

- 页面上某个源报错
- 你想让 AI 先定位问题，再决定删、换、修

直接复制：

```text
请使用伯乐Skill，帮我排查这个失败源：

- 失败源名称：[这里填名称]
- 失败 URL：[这里填网址]
- 当前现象：[这里填 404 / 403 / 空数据 / 解析失败 / 超时]

要求：
1. 先确认是源地址失效、页面结构变化、网络问题，还是解析器问题。
2. 如果原地址已经失效，请优先找稳定替代方案。
3. 如果页面结构变了，请尽量做稳一点的解析，不要只匹配一次性 DOM。
4. 如果这个源本身维护价值不高，可以建议直接删掉。
5. 修完后请告诉我：根因、改法、风险、验证结果。
```

## 模板 4：帮我做我的私人版本

适合场景：

- 你想把公共站点改成更贴合自己兴趣
- 比如偏向 Cursor、Claude Code、Codex、Trae、MCP、Skills

直接复制：

```text
请使用伯乐Skill，帮我把这个 AI 资讯雷达调整成更适合我的私人版本。

我的高优先主题：
- Cursor
- Claude Code
- Codex
- Trae
- MCP
- Skills / Agents

我的要求：
1. 默认层优先保留这些高信号主题。
2. 可以保留一部分更广义的 AI 行业新闻，但不要太吵。
3. 请区分“默认公开源”和“可选高级源”。
4. 默认公开流程保持免费、可 fork、可自动更新。
5. 如果你建议新增源，请说明为什么它值得长期追踪。
```

## 模板 5：帮我做 GitHub Pages 免费部署版

适合场景：

- 你 fork 了仓库
- 想让 AI 帮你一步步改成自己的站

直接复制：

```text
请使用伯乐Skill，帮我把这个项目整理成我自己的 GitHub Pages AI 资讯站。

目标：
1. 网址部署到我的 GitHub Pages
2. 保持公开仓库可运行
3. 默认不依赖任何付费 API
4. GitHub Actions 定时自动更新

请按这个顺序帮助我：
1. 检查仓库现状
2. 让我提供 GitHub 用户名和目标偏好
3. 调整默认信息源
4. 检查工作流与 Pages 配置
5. 生成和验证数据
6. 告诉我最终访问地址和后续维护方式
```

## 模板 6：帮我控制成本和高级源

适合场景：

- 你考虑接 `X API`
- 你考虑接 `AgentMail`
- 你不确定哪些会花钱

直接复制：

```text
请使用伯乐Skill，帮我检查这个项目的高级源成本和风险。

请重点分析：
1. 哪些能力默认免费
2. 哪些能力需要 token / key
3. token / key 本身是不是收费
4. 背后的服务是否收费，免费额度是多少
5. 哪些高级源适合默认关闭
6. 怎样保证公共仓库不泄露隐私和秘密

请最后把结果分成：
- 默认免费层
- 可选免费试用层
- 明确可能收费层
- 不建议公共默认开启层
```

## 模板 7：帮我做一次仓库体检

适合场景：

- 你担心仓库越来越重
- 想让 AI 检查归档、工作流、失败源、页面状态

直接复制：

```text
请使用伯乐Skill，帮我对这个 AI 资讯雷达做一次维护体检。

请检查：
1. 失败源和零产出源
2. 源池是否过宽或过窄
3. archive.json 等数据文件是否过大
4. GitHub Actions 是否有不必要的开销
5. GitHub Pages 页面文案和统计是否准确
6. 是否有可以删掉、收窄或改成更稳方案的地方

请输出：
- 最高优先级问题
- 建议立即修改的项
- 可选优化项
- 不建议现在动的项
```

## 你也可以让 AI 先向你提问

如果你不想一次性把需求写完整，可以复制这一段：

```text
请使用伯乐Skill，但先不要直接改代码。先问我 5 个以内最关键的问题，确认我的目标、偏好、信息源范围、预算限制和是否接受高级私有源，然后再给我一个最小可执行方案。
```

## 推荐你每次都附带的约束

下面这段可以追加在任何模板后面：

```text
补充约束：
- 优先小改，不要无关重构
- 优先官方 RSS / Atom / feed / changelog
- 公开默认层尽量免费
- 私人高级源和公共默认源要分开
- 不要提交 secrets、token、cookie、私有 OPML、私有邮箱内容
- 改完后请告诉我改动文件、验证结果、剩余风险
```

## 最后给你的建议

如果你以后不知道该复制哪一段，就先用这句：

```text
请使用伯乐Skill，先问我目标和信息源清单，再用最省钱、最稳定、最容易长期维护的方式帮我改这个 AI 资讯雷达。
```

这句最通用，基本不会跑偏。
